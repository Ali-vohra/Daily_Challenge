Taking the Drone Less Traveled

Sometimes, computer scientists face problems with few, if any, solutions. Other times, problems seem to have too many solutions. For a computer scientist, it can be surpassingly tricky to design algorithms for problems that have a huge number of possible solutions!

When an algorithm has to navigate the real world, it's common to encounter explosions in the number of possible solutions. Self-driving cars obviously have this problem, and so does your "friendly" autonomous surveillence-and-same-day-delivery drone, the Eye of Amazoo.

The Eye of Amazoo

Take a look at the situation described below. (You can run the program by pressing the Run Code button.)

Run Code

There's only one sequence of three instructions that can get the drone to the red barn #1. The drone must go East/Right twice, and then go South/Down once.

In a more open map, there are suddenly three different paths to the orange barn #2. (You can modify this program by dragging tiles from the top row onto the program in the second row.)

Run Code

Every solution uses the same three steps: two moves to the East/Right and one to the South/Down. But the drone's move to the south can happen first, second, or third.

Another kind of open-endedness comes from the drone needing to take extra steps. If we update the rules to say

  1. the drone must reach the barn and
  2. the drone can only reach a barn at the very last step,

then there are suddenly two different programs that solve this puzzle in exactly five steps:

Run Code

The map is the same as the map from the first step, so the overall movement is the same: the drone must go East/Right twice, and then South/Down once.

The rules mean that the "South/Down" instruction can only be the very last instruction: it's against the rules to visit the yellow barn #3 more than once. The extra two instructions must be an extra movement to the left, followed by an extra movement to the right. The drone is not permitted to touch any barns — including the pink barn #6 — before its final delivery.

There are seven moves which can be North, East, South, West (N, E, S, W). These are:

  1. E.

  2. N, E, or W.

  3. Reverse of 2.

  4. N, E, or W.

  5. Reverse of 4

  6. E.

  7. S.

Our only choices are in 2 and 4 (edit: corrected typo), each of which has three choices and each of which is independent (we end up in the same spot one move later no matter which choice we take). Because the choices are independent, we don't have to follow each pathway but can simply multiply them together.

Number of sequences:

n=3×3=9n = 3 \times 3 = 9n=3×3=9

% Solution to: https://brilliant.org/daily-problems/drone-paths/
% Taking the Drone Less Traveled
% 2020-04-23

clear
Options = [-1,1,-1i,1i];
Counter = 0;
PositionStart = 1;
Steps = zeros(4^4,4);
AllowedPositions = 'Position == 0 || Position == 1 || Position == 2 || Position == 1 + 1i';

for ind1 = [-1,1,1i]
    Position = PositionStart + ind1;
    for ind2 = Options
        Position = PositionStart + ind1 + ind2;
        if eval(AllowedPositions)
            for ind3 = Options
                Position = PositionStart + ind1 + ind2 + ind3;
                if eval(AllowedPositions)
                    for ind4 = Options
                        Position = PositionStart + ind1 + ind2 + ind3 + ind4;
                        if Position == 1
                            Counter = Counter + 1;
                            Steps(Counter,:) = [ind1,ind2,ind3,ind4];
                        end
                    end
                end 
            end
        end
    end
end

Steps(Counter+1:end,:) = [];
UniqueSequences = unique(Steps,'rows');

"""
DSF-type solution of:
    https://brilliant.org/daily-problems/drone-paths/
"""

def get_all_valid_move_lists(start_pos, dir_code_list, max_len, is_pos_valid_fun, is_end_pos_fun):
    """
    returns the list of all valid move sequences using DFS approach
    """
    def is_list_valid(start_pos, move_list, max_len, is_pos_valid_fun, is_end_pos_fun):
        res = True
        cur_pos = start_pos
        for cur_dir in move_list:
            if not is_pos_valid_fun(cur_pos):
                res = False
                break
            cur_pos = make_move(cur_pos, cur_dir)
        if res and len(move_list) == max_len:
            if not is_end_pos_fun(cur_pos):
                res = False
        return res

    def make_move(in_pos, dir_code):
        if dir_code == 'N':
            res_pos = (in_pos[0], in_pos[1]+1)
        elif dir_code == 'S':
            res_pos = (in_pos[0], in_pos[1]-1)
        elif dir_code == 'E':
            res_pos = (in_pos[0]+1, in_pos[1])
        elif dir_code == 'W':
            res_pos = (in_pos[0]-1, in_pos[1])
        return res_pos
    sol_list = []

    def inner(cur_move_list):
        nonlocal sol_list
        if is_list_valid(start_pos, cur_move_list, max_len, is_pos_valid_fun, is_end_pos_fun):
            if len(cur_move_list) == max_len:
                sol_list.append(cur_move_list)
            else:
                for new_dir in dir_code_list:
                    tmp_move_list = cur_move_list.copy()
                    tmp_move_list.append(new_dir)
                    inner(tmp_move_list)
    inner([])
    return sol_list

def get_start_pos():
    """
    returns the starting position as in
    https://brilliant.org/daily-problems/drone-paths/
    """
    return (0, 1)

def is_valid(in_pos):
    """
    checks if given position is valid
    (i.e. it is not outside the board and is not one of the barns)
    as in
    https://brilliant.org/daily-problems/drone-paths/
    """
    res = True
    bran_pos_set = {(0, 0), (0, 2), (1, 0), (2, 0), (3, 1), (2, 2)}
    pos_x = in_pos[0]
    pos_y = in_pos[1]
    if not (0 <= pos_x < 4 and 0 <= pos_y < 3):
        res = False
    elif in_pos in bran_pos_set:
        res = False
    return res

def is_end_pos(in_pos):
    """
    checks if given position is the end barn in
    https://brilliant.org/daily-problems/drone-paths/
    """
    return in_pos == (2, 0)

DIR_CODE_LIST = ['N', 'S', 'E', 'W']
SOL_LIST = get_all_valid_move_lists(get_start_pos(), DIR_CODE_LIST, 7, is_valid, is_end_pos)
for cur_sol in SOL_LIST:
    print(', '.join(cur_sol))
print(f"Number of solutions: {len(SOL_LIST)}")

    adMat[1][2] = adMat[2][1] = 1;//bc & cb
    adMat[1][3] = adMat[3][1] = 1;//bd & db

    int[][] adMatTemp = adMat;
    int numberOfMoves = 7;
    for (int i = 2; i<numberOfMoves; i++) {
        adMatTemp = matrixMul(adMatTemp, adMat);
    }
    int result = adMatTemp[0][3]; //start at a finish at d  
    System.out.println(result);
}

public static int[][] matrixMul(int[][] m1, int[][] m2) {
    int[][] result = new int[m1.length][m1.length];

    for(int i = 0; i < m1.length; i++) {
        for(int j = 0; j < m1.length; j++) {
            for(int k = 0; k < m1.length; k++) {
                result[i][j] += m1[i][k] * m2[k][j];
            }
        }
    }
    return result;
}

The first instruction must be moving one space to the right in order for the drone to not touch any barns. The last two instructions must be moving right and then moving down to the pink barn which is the drone's destination. So we have R _ _ _ _ R D. Now we have four spaces left for extra moves. This can be split into two pairs of moves that cancel each other out e.g. up, then down immediately after. So we can think of it as R { _ _ }{ _ _ } R D where the brackets hold one pair of moves that cancel out.

Once we move one to the right for our first move, we can see that square that we are currently on as our 'middle' or 'home base' square that we must return to after each pair of moves. There are three options that will bring us back to the home base square.

Up, down

Right, left

Left, right

Then we are left with one more possible pair of moves than cancel out. You apply the three options again.

So for the first pair of moves, you have three choices, and after you choose one of them, you have three choices again. For each first pair of moves that you choose, there are three cases for the second.

So there are 3, the first pair choices, times 3, the second pair choices, which is 9 different pathways to take.

The following is a Perl language algorithm to find the answer. Its recursive meaning it follows a Depth First Search:

$seq_tracker = 0;
@curr_pos = (1,0);
prob042320(\@curr_pos,0);
print "$seq_tracker\n";
exit;

sub prob042320{
    my ($curr_ref,$moves) = @_;
    my @curr_pos = @{$curr_ref}; #2 items. y, x position
    my @graph = ([X,0,X,X],
                 [0,0,0,X],
                 [X,X,0,X]); 
    my @moves = ("U","D","L","R");

    if($moves > 7){#cannot have more than 7 moves
        return;
    }
    if(($curr_pos[0] < 0) || ($curr_pos[1] < 0)||
       ($curr_pos[0] > 2) || ($curr_pos[1] > 3)){#exit if out of bounds
        return;
    }
    if($graph[$curr_pos[0]][$curr_pos[1]] eq "X"){#exit if pos is on an X
        return;
    }
    if(($curr_pos[1] == 2)&&($curr_pos[0] == 2)){#exit position
        if($moves == 7){
            print "found seq\n";
            $seq_tracker++;
            return;
        }
        else{#exit if it reached endpoint but not at 7 moves
            return;
        }
    }

    foreach my $next (@moves){
        my @new_pos = @curr_pos;
        if($next eq "U"){
            $new_pos[0]++;
        }
        elsif($next eq "D"){
            $new_pos[0]--;
        }
        elsif($next eq "L"){
            $new_pos[1]--;
        }
        elsif($next eq "R"){
            $new_pos[1]++;
        }
        prob042320(\@new_pos,($moves+1));
    }

}

First of of all, the drone's first move has to be right (see blue), and since it cannot arrive at the pink(6) barn early, it must end on that square as it its last move. The only option for that last move is down (see pink) because all other adjacent squares are either inaccessible or barns. The predecessor to the last move we see has to be from the right as well (see green), leaving us with four areas of deviation: Notice that where the drone has to move to in the beginning and where the drone has to move from on the second to last move are the same, meaning that our moves have to contradict each other to get us back to that square (granted that we can't choose not to move). There are three possibilities as follows: (right,left),(left,right),(up,down). We cannot do (down,up) because it runs us into a barn. With 3 different options in 2 slots (as each conjugate pair takes up two moves), we have 3^2 = 9 different options. Note: It is only possible to simplify the remaining set of actions as conjugate pairs and not (right,right,left,left) as opposed to (right,left,right,left) because every move in one direction from the intermediate tile leads us either to the edge of the board or to a barn.

All 7 step-paths from current point to the pink barn must begin with the one-step sequence "East" and finish with the two-step sequence "East then South".

So we just need to count the number of possible 4-step paths that starts from the middle point (north of the blue barn) and returns to it.

From that middle point, we start with 3 possibilities: we may go either West, North or East, and on each direction we are are forced to go back immediately to the same middle point.

Back there, we are given again 3 possibilities, as we are free to choose again either West, North or East, and then forced to go back.

That makes 3x3=9 possible paths.

Moving back one step, the sixth move must consist of moving to the square above the pink barn, and there is only one way to do this. Thus, the first move, the sixth move, and the seventh move are completely determined.

This leaves you freedom on moves 2, 3, 4, and 5. On round 2 you have three options: East, North, West. Then on round 3 you need to do the opposite of whatever you did in round 2.   • Based on the starting position, we know the first move has to be an East, since it's the only legal move.
  • We also know that final move must be a South, from the cell directly north of the pink barn (there's no other way to reach the pink barn).
  • We then also know that the second to last move must be an East from the cell directly NW of the pink barn.

So we know the moves are: E ? ? ? ? E S.

The best way to see this is to work backwards. For the sake of labeling, call the initial space A, the space to the right of A call B, the space north of B call C, and the space to the east of B call D. If we are going to get to the pink barn in 7 steps, we must get to point D in 6 steps. For this to happen, we have to get to point B in 5 steps. But, since our first move has to be from A to B, this last criterion is equivalent to getting from B back to B in exactly 4 steps. Starting at B, we have three choices for a move. If we choose, say, to move to C, we now have to get back to B in exactly 3 steps. But, if we've moved to C, our next move has to be back to B. Therefore, success in this case is equivalent to getting from B back to B in two steps. There are clearly 3 ways for this to occur (B to A to B, B to C to B, or B to D to B). So, each of the three choices has three successful outcomes. So, the total number of successful outcomes is 3*3 = 9.
