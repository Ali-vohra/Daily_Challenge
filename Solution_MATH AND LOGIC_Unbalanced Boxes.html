    Solution

    Explanation of steps:

    • Kaitlin doesn't have most tiles, has at least a tile which is a multiple of 333.

    • Sum of Kevin's tiles is 13\dfrac{1}{3}31​ of Kaitlin, hence Kaitlin must have tiles of big magnitude.

    • However, 888 and 777 aren't multiples of 333. Using 333 and some other tile needs a copy of that tile. And, by little observation, 666 and 555 don't work. Hence we use 6×4=246\times4 =246×4=24.

    • We need the sum to be 111111. Hence, we use 6,4,16, 4, 16,4,1, whose product is also 242424.

    • Since Kevin has 13\dfrac{1}{3}31​ of Kaitlin, he only has 888 (we don't have 444 left to use with a 222).

    • We are left with four tiles and there is no restriction so we can put them in Conor's box.

    2+3+5+7=172+3+5+7=\boxed{17}2+3+5+7=17​

    # Unbalanced Boxes
# Python

# set up variables
#  p is the list of all possible permutations of tiles 1 through 8
#  g is the possible number of tiles for Conor, Kevin, and Kaitlin
from itertools import permutations
p = list(permutations(range(1, 9)))
g = [(4, 1, 3), (4, 3, 1), (1, 4, 3), (3, 4, 1), (
    5, 1, 2), (5, 2, 1), (1, 5, 2), (2, 5, 1)]
res = []

# go through each possible combination of p and g
for a in range(0, len(p)):
    for b in range(0, len(g)):
        # get lists of Conor's, Kevin's, and Kaitlin's tiles
        Co = []
        Ke = []
        Ka = []
        for c in range(0, g[b][0]):
            Co.append(p[a][c])
        for c in range(g[b][0], g[b][0] + g[b][1]):
            Ke.append(p[a][c])
        for c in range(g[b][0] + g[b][1], g[b][0] + g[b][1] + g[b][2]):
            Ka.append(p[a][c])
        # find the sum of Kevin's tiles
        KeS = 0
        for c in range(0, len(Ke)):
            KeS += Ke[c]
        # find the sum and product of Kaitlin's tiles
        KaS = 0
        KaP = 1
        for c in range(0, len(Ka)):
            KaS += Ka[c]
            KaP *= Ka[c]
        # if the combination fits the requirements, sort and store it
        if 3 * KeS == KaP and KaS == 11:
            s = (sorted(Co), sorted(Ke), sorted(Ka))
            if s not in res:
                res.append(s)
# display the results
for a in range(0, len(res)):
    print("Conor's tiles: " + str(res[a][0]))
    print("Kevin's tiles: " + str(res[a][1]))
    print("Kaitlin's tiles: " + str(res[a][2]))

    Lets go step by step with each information given.

    From Conor's statement, given all 8 tiles are used and everyone has different number of tiles, then there are two possibilities for the number of tiles of each person:

    • 1, 2, and 5 (P1)

    • 1, 3 and 4 (P2)

    From Kevin's statement, since his sum is one third of the product of Kaitlin's tiles, Kaitlin must have tile 3 or 6, or both of them.

    From Kaitlin's statement, and the two possibilities above, Kaitlin either has 2 or 3 tiles (She can't have 1 tile since her sum is 11 and she can't have 5 or 4 tiles since she doesn't have the most tiles).

    Let's first assume that Kaitlin has 2 tiles. Then the only possibilities can be tiles 3 and 8 or tiles 5 and 6 (note that the sum is 11). If Kaitlin's tiles are 3 and 8, then the sum of the Kevin's tiles is 3×83=8.\frac{3 \times 8}{3} = 8.33×8​=8. But this is not possible since Kevin can't have 1 tile (note tile 8 already belongs to Kaitlin and can't have 5 tiles (There is no set of 5 remaining tiles whose sum is 8).

    The same reasoning applies to the case where Kaitlin has tiles 5 and 6.

    • 1, 2, and 5
    • 1, 3, and 4

    • 1, 3, 7
    • 1, 4, 6
    • 2, 3, 6
    • 3, 8
    • 5, 6

    • 7×3×1=21
    • 6×4×1=24
    • 6×3×2=36
    • 8×3=24
    • 5×6=30

    36÷3=12 and 30÷3=10, and the greatest single tile is 8.
