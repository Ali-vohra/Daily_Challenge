Fill the Pyramid

Grid Fill puzzles like the one below are similar to Sudoku puzzles in that they ask you to place numbers in squares in a certain way, but the similarities end there.

If you're familiar with how these work, jump straight to today's problem. If you want to look through some worked examples, though, keep reading.

One of the differences between these puzzles and Sudoku is that a Grid Fill solution can have a number appearing more than once in the same row or column. The goal is to drag number tiles from the bank to squares on the grid so the rows and columns sum to the indicated values.

For example, a solution to the puzzle below will have numbers in the top row with a sum of 3. Note that the middle square of that row is gray and will not accept a number tile, so its effective value is zero.     I solved this one like Stefan Smith did, but it's a nice little problem to play with if you are learning Constraint Programming. Here's how to tackle it in Google's OR-Tools in Python. It was particularly useful to learn about the AddElement constraint that lets us map index values to possible options. 

    from ortools.sat.python import cp_model
model = cp_model.CpModel()
options = [1, 2, 2, 3, 3, 3, 4, 4, 5]
sums = [
    ((0, 1, 2, 3, 4), 16),
    ((5, 6, 7), 8),
    ((1, 5), 5),
    ((2, 6, 8), 6),
    ((3, 7), 7),
]
index_variables = [
    model.NewIntVar(0, len(options)-1, str(i))
    for i, _ in enumerate(options)
]
puzzle_variables = [
    model.NewIntVar(0, 1000, f"z_{i}")
    for i, _ in enumerate(options)
]
for idx, p in zip(index_variables, puzzle_variables):
    model.AddElement(idx, options, p)
model.AddAllDifferent(index_variables)

for idx_list, val in sums:
    model.Add(sum(puzzle_variables[i] for i in idx_list) == val)

    Here's my reasoning:

    Step 1: The 5 can only go into the column that adds up to 7 and the tiles needed are 5 and 2.

    Why?

    a. If the 5 tile were to go into the column that adds up to 5, we'd need a zero tile (5=5+0), which we do not have.

    b, If the 5 tile were to go into the column that adds up to 6, we'd also need a zero tile (6=5+1+0), which we don't have.

    Step 2: The only way we can make a 6 in the column that adds up to 6 is to use tiles 1, 2 and 3.

    Why?

    a. 6=0+1+5, which doesn't work because we have no zero tiles and no 5 tile left

    b. 6=1+2+3, which does work because we have the required tiles.

    c. Any other combination once again requires us to have a zero tile.

    Step 3: The only way to make a 5 in the column that sums up to 5 is to use either a 1 and 4 tile (5 = 1+4) or a 2 and 3 tile (5=2+3), but at this point we have no more 1 tiles or 2 tiles, since these were used up in steps 1 and 2.

    Also note that in order to reach this conclusion, we do not need to consider the order in which the tiles are placed into each column.
