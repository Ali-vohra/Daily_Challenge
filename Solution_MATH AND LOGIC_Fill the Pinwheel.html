Fill the Pinwheel

Grid Fill puzzles like the ones below are similar to Sudoku puzzles in that they ask you to place numbers in squares in a certain way, but the similarities end there.

If you're familiar with how these work, jump straight to today's challenge. If you want to look through some worked examples, though, keep reading.

One of the differences between these puzzles and Sudoku is that a Grid Fill solution can have a number appearing more than once in the same row or column. The goal is to drag number tiles from the bank to squares on the grid so the rows and columns sum to the indicated values.

For example, a solution to the puzzle below will have numbers in the top row with a sum of 3.3.3. Note that the middle square of that row is gray and will not accept a number tile, so its effective value is zero. By clicking the right arrow below the puzzle, you can view a demonstration of a puzzle solution complete with the underlying logic. If you need to see a step again, click the left arrow.

tiles = [0, 0, 1, 1, 1, 2, 2, 2, 3, 3, 3, 4, 4]
solutions = []
cache = set()

def calc(n, grid, used):
    if n == 12:
        if grid not in solutions:
            print(grid)
            solutions.append(grid.copy())
        return
    else:
        # starting permutation algorithm
        # accelerated by early comparing the grid-sums criteria
        for i, t in enumerate(tiles):
            if i in used:
                continue
            grid[n] = t
            if n == 4:
                # luckily digits repeat otherwise 13! seems a bit too much to work out
                if tuple(grid[:5]) in cache:
                    continue
                else:
                    cache.add(tuple(grid[:5]))
                if sum(grid[1:5]) != 8:
                    continue
            if n == 6:
                if sum(grid[5:7]) != 4:
                    continue
            if n == 8:
                if grid[3] + grid[8] != 8:
                  continue
            if n == 9:
                if grid[0] + grid[4] + grid[6] + grid[9] != 8:
                  continue
            if n == 10:
                if sum(grid[7:11]) != 8:
                    continue
            if n == 11:
                if grid[2] + grid[5] + grid[7] + grid[11] != 8:
                  continue
            used.add(i)
            calc(n+1, grid, used)
            used.remove(i)


calc(0, [0]*12, set())
print(len(solutions))

Reason

  • In the middle, both the '4' tiles as there are 2 tiles and they need to add to 8. Thus 4×2=84 \times 2=84×2=8.
  • Again, around the middle horizontally, 2 tiles should add to 4. As we already used both the tiles, we can either use '3' or '1' tile together or '2' and '2' tile together as 3+1=2+2=4. Due to this, 2 unique solutions arise.
  • With each 1 unique solution, we can obtain 3 more unique solutions by simply seeing the symmetry of the pinwheel. We have 4 corners, so using combinations and rotations, we can create 3 extra solutions.
  • Thus, as we already created 2 unique solutions with the 2nd point, and with each 1 solution more 3 solutions, so in total, we get 1+3+1+3=8 solutions.

from itertools import permutations

#       A
# B C 4 D
#   E   F
#   G 4 H I
#   J
print('      A\nB C 4 D\n  E   F\n  G 4 H I\n  J\n')
tiles = [0, 0, 1, 1, 1, 2, 2, 2, 3, 3, 3]
solutions = set()

for p in permutations(tiles, 10):
    A, B, C, D, E, F, G, H, I, J = p
    tests = [
        (B + C + 4 + D) == 8,
        (E + F) == 4,
        (G + 4 + H + I) == 8,
        (C + E + G + J) == 8,
        (A + D + F + H) == 8,
    ]
    if all(tests) and p not in solutions:
        solutions.add(p)
        print(f'      {A}\n{B} {C} 4 {D}\n  {E}   {F}\n  {G} 4 {H} {I}\n  {J}\n')
print(f'ANSWER: {len(solutions)}')

class Board(object):
    def __init__(self, row_data, column_data, shape):
        self.num_solutions = 0
        self.board_shape = shape
        self.needed_row_sums = row_data
        self.needed_col_sums = column_data

        self.row_has_specific_sum = list(i != 999 for i in row_data)
        self.col_has_specific_sum = list(j != 999 for j in column_data)

        # Create a list of tuples with all the valid coordinates of
        # numbers we can place. This doesn't really optimize anything,
        # it just makes it easier to code
        self.coords = []
        self.num_tiles_in_row = [0]*max(len(board_shape[i]) for i in range(0, len(board_shape)))
        self.num_tiles_in_col = [0]*len(board_shape)

        for i in range(0, len(board_shape)):
            for j in range(0, len(board_shape[i])):
                if shape[i][j] == ' ':
                    self.coords.append((i, j))
                    self.num_tiles_in_row[i] += 1
                    self.num_tiles_in_col[j] += 1

    def print_solution(self):
        print("Solution:")
        for i in range(0, len(self.board_shape)):
            for j in range(0, len(self.board_shape[i])):
                print(self.board_shape[i][j], end=' ')
            print()
        print()

    def sum_obtainable(self, row, col, max_tile_val):
        """ Determines if the sums in both the row and the column are obtainable. 
            The case where there are no tiles left to place in a row / col is handled. """
        row_obtainable = True
        col_obtainable = True

        if self.row_has_specific_sum[row]:
            row_obtainable = (self.needed_row_sums[row] <= max_tile_val*self.num_tiles_in_row[row])
        if self.col_has_specific_sum[col]:
            col_obtainable = self.needed_col_sums[col] <= max_tile_val*self.num_tiles_in_col[col]

        return (row_obtainable and col_obtainable)

    def solve(self, current_index, tiles):
        """ standard depth-first, guess-and-check solver """
        if current_index == len(self.coords):
            self.print_solution()
            self.num_solutions += 1
        else:
            # place a tile down
            i, j = self.coords[current_index]  # make code more readable

            # The only tiles we can place are:
            # A) the tiles that exist
            # B) the tiles that don't make the sum go over the requested sum.
            # This assumes that all tiles are >= 0, which is true in this challenge.
            # The max tile value is hardcoded as 4 to prevent computing it over and over.
            for n in range(min(self.needed_row_sums[i], self.needed_col_sums[j], 4), -1, -1):
                if tiles[n] > 0:
                    # we found a valid tile we can place!
                    tiles[n] -= 1
                    self.needed_row_sums[i] -= n
                    self.num_tiles_in_row[i] -= 1

                    self.needed_col_sums[j] -= n
                    self.num_tiles_in_col[j] -= 1

                    self.board_shape[i][j] = n

                    # Another optimization is seeing if the sum is obtainable within the current tile setup;
                    # if the (needed sum - current sum)/(number of tiles remaining in that row or col) >
                    # (max tile value that we can pick from), then the sum is unobtainable.

                    if self.sum_obtainable(i, j, max(t for t in tiles.keys() if tiles[t] > 0)):
                        self.solve(current_index + 1, tiles)

                    # give the tile back
                    self.board_shape[i][j] = ' '
                    self.needed_col_sums[j] += n
                    self.needed_row_sums[i] += n
                    self.num_tiles_in_col[j] += 1
                    self.num_tiles_in_row[i] += 1
                    tiles[n] += 1
