def eval(n, grid, visited, solutions):
    for i, tile in enumerate([1, 1, 2, 3, 3, 6, 7]):
        if i in visited:
            continue
        grid[n] = tile
        if n == 2:
            if not sum(grid[:3]) == 10:
                continue
        if n == 5:
            if not grid[0] + grid[3] + grid[5] == 10:
                continue
        if n == 6:
            if not (grid[5] + grid[6] + 5 == 10 and
                grid[2] + grid[4] + grid[6] == 10):
                continue
        visited.append(i)
        eval(n+1, grid, visited, solutions)
        visited.remove(i)

    if n == 7 and grid not in solutions:
        print(grid[:3])
        print([grid[3], 0, grid[4]])
        print([grid[5], 5, grid[6]])
        print()
        solutions.append(grid.copy())

There are five odd numbers in the set. If we won't put one odd number in the asked position, then the sum of either left or right column will be odd, which we know isn't possible. So, the asked number is an odd number.

Let's start step by step.

Now, sum of last row is 10, which is possible only when a 3 and a 2 is placed there. Let it be like below as order won't matter.

Now, sum of left column is 10, which is possible only for a 7 and a 1 now. If 7 is placed in the first cell, then we will need a 1 on X(as 3 means 0, which we don't have) and then a 2 on top right cell, but we have only one two, so 7 is in (1, 2).

Now, sum of top row is 10, which is only possible for for a 3 and a 6. Since we know that X is odd, so X=3\boxed{X = 3}X=3​

With the reasoning, we have proved that there are only two symmetric solutions.

Step 1: We need a sum of 5 in two tiles on the bottom row. Tiles 6 and 7 are too high, tile 1 is too low since we don't have a 4. Hence, we have 2 + 3, in some order (turns out the order isn't important since the solution will work both ways).

Step 2: We need a sum of 10 in three tiles on the top row. 7 can't be used since we don't have a 2 anymore. Using two 1's isn't an option since we don't have an 8. Hence, the only sum available is 1 + 3 + 6, in some order.

Step 3: On the left column, we need a sum of 7 in two tiles. We can't use the 7 tile because would mean a sum of 7 in only one tile. So we place the 1 tile in the middle, repositioning the 6 in the top row accordingly. Hence, we have 3 + 1 + 6.

Step 4: One tile left (7) and one place left. We reposition the 1 tile in the top row accordingly. Finally, we have 2 + 7 + 1.

Conclusion: The number placed at the top of the central column is the number 3.
