Even to Odd

What if we had an equation where the numbers on one side of the equation didn't change but the arithmetic operations among them did? As the operations changed, the result on the other side would change, too.

To solve an operator search puzzle, you need to find the operations that make the equation have a desired result. Keep reading to see a few examples with strategies used to solve them, or jump ahead to today's challenge if you feel ready.

Let's start by looking at a puzzle where only addition and subtraction are allowed. Is it possible to replace every square with either a +++ or a - to make the equation true:

2  3  5  6=10? 2\ \square\ 3\ \square\ 5\ \square\ 6 = 10? 2  3  5  6=10?

Let's develop some reasoning we can extend to more complex puzzles. The maximum value we can get is 2+3+5+6=16,2 + 3 + 5 + 6 = 16,2+3+5+6=16, which is too large by six. What if we subtract six instead of adding it? That would give us 2+3+56=4,2 + 3 + 5 - 6 = 4,2+3+56=4, which is too small... again, by six. Changing the sign immediately before the 666 decreased the result by 26=12.2 \times 6 = 12.26=12.

Let's change it back to 2+3+5+6=16,2 + 3 + 5 + 6 = 16,2+3+5+6=16, and note that we want to decrease the result by 1610=6.16 - 10 = 6.1610=6. Changing the +++ sign in front of the 333 should do it, since 23=6.2 \times 3 = 6.23=6. Let's try it:

23+5+6=10. 2 - 3 + 5 + 6 = 10. 23+5+6=10.

This indeed works. Feel free to play with an interactive version of the same puzzle below, clicking on the operators to change them. Does the parity of the result ever change?

With more operations, things get more complicated. If we replace each square with a +, , +,\ -,\ \times+, ,  or a ,\div,, can we make this equation true:

8  4  2  12  4=3? 8\ \square\ 4\ \square\ 2\ \square\ 12\ \square\ 4 = 3? 8  4  2  12  4=3?

Let's think about the parities of the numbers. The result is odd, while every other number is even. The only way we can get an odd number from two even numbers is by dividing them, so at least one of the squares must be replaced with a \div symbol. The only pair of consecutive numbers with an odd quotient are 121212 and 4,4,4, so let's put a \div symbol between them. Multiplication and division are performed before addition and subtraction according to the order of operations, so we can go ahead and simplify 124:12 \div 4:124:

8  4  2  124=38  4  2  3=3.\begin{aligned} 8\ \square\ 4\ \square\ 2\ \square\ 12 \div 4 & = 3 \\ 8\ \square\ 4\ \square\ 2\ \square\ 3 & = 3. \end{aligned} 8  4  2  1248  4  2  3=3=3.

Our puzzle looks a little easier now that we've simplified it. We know that 8=42,8 = 4 \times 2,8=42, so 842=0:8 - 4 \times 2 = 0:842=0:

842  3=30  3=30+3=3.\begin{aligned} 8 - 4 \times 2\ \square\ 3 & = 3 \\ 0\ \square\ 3 & = 3 \\ 0 + 3 & = 3. \end{aligned} 842  30  30+3=3=3=3.

We can then work backwards and substitute symbols for the original equation's squares to make it balance:

842+124=3. 8 - 4 \times 2 + 12 \div 4 = 3. 842+124=3.

Finally, let's look at a puzzle allowing a new operation: concatenation. When we concatenate a pair of numbers, we slide them together. For example, 23=232 || 3 = 2323=23 and 1280=1280.12 || 80 = 1280.1280=1280. Can we make the equation true if we're limited to +,,+, -,+,, and || operators:

2  2  2  2=44? 2\ \square\ 2\ \square\ 2\ \square\ 2 = 44? 2  2  2  2=44?

There's no way we can get a result as large as 444444 using only addition and subtraction, so we need to use at least one || symbol somewhere. While 22=222 || 2 = 2222=22 is closer, it's still too small. Meanwhile, 222=2222 || 2 || 2 = 222222=222 is far too large, so we can't have two or more || symbols in a row. Noticing that 22=222 || 2 = 2222=22 is half of 44,44,44, we can add two copies of 22:2 || 2:22:

2  2+2  2=44. 2\ ||\ 2 + 2\ ||\ 2 = 44. 2  2+2  2=44.

    How to find the solution?\text{How to find the solution?}How to find the solution?

    First, as 4,2,64,2,64,2,6 are even numbers, there is no way that you can find an odd result without using division.

    Therefore, there must be at least one division.

    Possible divisions that have an odd result:

    • 426=7 4 || 2 \div 6 = 7426=7

    • 62=36 \div 2 = 362=3

    • 262=132 || 6 \div 2 = 13 262=13

    Let's work on each scenario.

    Scenario 1: 426=7\text{Scenario 1: } 4 || 2 \div 6 = 7 Scenario 1: 426=7

    In order to bring 777 down below zero, the only way is through subtraction (division is not possible since we can't use 2 orders of operations, which includes subtraction, at the same blank)

    However, 2<72 < 72<7 so this is impossible.

    Scenario 2: 62=3\text{Scenario 2: } 6 \div 2 = 3 Scenario 2: 62=3

    In this scenario, the best way to bring the result down below zero is through subtraction, or both subtraction and division (only division is not possible, as stated above)

    Only subtraction: 4262=14 - 2 - 6 \div 2 = -14262=1 (Acceptable)

    Both subtraction and division: 4262=14 \div 2 - 6 \div 2 = -14262=1 (Acceptable) (We consider only 1 equation, as 2 can't divide 6)

    Last Scenario: 262=13\text{Last Scenario: } 2 || 6 \div 2 = 13 Last Scenario: 262=13

    Like the first scenario, the only way is through subtraction.

    4262=94 - 2 || 6 \div 2 = -94262=9 (Acceptable)

    Conclusion\text{Conclusion}Conclusion

    • In the second scenario, the final result is 1-11 whereas the final result of the last scenario is 9-99.

    The absolute value of 1-11 is 1=1|-1| = 11=1, whereas the absolute value of 9-99 is 9=9|-9| = 99=9

    Therefore, 9-99 is the only possible odd value of the equation, which has the highest absolute value.

    Using binary to mean two operands, three binary operations are called for in this problem. Let's label the operations from left to right as: (A), (B) and (C).

    The initial problem is: 4 (A) 2 (B) 6 (C) 2 = ( most negative, odd integer)

    Looking at all 125 possibilities is undesirable because some possibilities simply don't make sense. For example, (A) = (B) = (C) = addition, can't possibly yield the correct result.     expression 1

    expression 2

    { 4 (A) 2 } { 6 (C) 2 } , where B = subtraction

    expression 3

    The 75 possibilities in these three expressions is better than the original 125 possibilities and brute-force could be used here, but the requirement that the result be odd enables a further reduction in possibilities. The only way to get an odd number given even operands is to use division. We can now require one of the remaining two operations to be division which narrows the possibilities to six expressions:

    INVALID_BRANCH = -1
CONTINUE_NEXT_BRANCH = 0
SOLUTION_FOUND = 1


def recurse(numbers, total, target, index=0):
    if total > target:
        for i in range(index, len(numbers)):
            result = total - 2 * numbers[i]
            status, branch = recurse(numbers, result, target, i + 1)

            if status == INVALID_BRANCH:
                break
            elif status == CONTINUE_NEXT_BRANCH:
                pass
            else:
                branch[i] = '-'
                return status, branch

        return CONTINUE_NEXT_BRANCH, None

    elif total == target:
        return SOLUTION_FOUND, ['+'] * len(numbers)

    else:
        return INVALID_BRANCH, None


def main():
    # 1 + 5 - 8 + 11 - 13 - 15 - 17 + 23 + 88 - 102 + 122 = 95
    numbers = [1, 5, 8, 11, 13, 15, 17, 23, 88, 102, 122]
    target  = 95

    status, solution = recurse(sorted(numbers), sum(numbers), 95)
    if status == SOLUTION_FOUND:
        print('Solution:', ' '.join(f'{x[0]} {x[1]}' for x in zip(solution, numbers)), '=', target)
    else:
        print('No solution found.')


if __name__ == '__main__':
    main()
