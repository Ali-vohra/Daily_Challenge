How Many Paths?

How many different ways are there to get from point A to point D, assuming no path ever traces back over any part of itself?

We could try to draw all of the paths separately one by one, but that might take a while. Keep reading to see one way we can answer this question more quickly, or skip straight to today's challenge for a slightly different problem.

Working backwards can be a helpful problem-solving strategy. Any path that ends up at point D must have gone through either point B or C just beforehand. We also know that no path can cover the entire arc from B to C since getting back to point D would force the path to trace back over part of itself.

This means we can completely determine any path ending at point D by seeing how the path gets to point B or C first. So, let's simplify the diagram by getting rid of the arc from B to C.

Symmetry is often nice to use when counting, and this problem is no exception. Let's focus on finding the number of paths that go from A to B without backtracking. This will be the same as the number of paths from A to C, so it'll save us some counting.

To go from A to B without backtracking, we can either go straight from A to B (AB), or go from A to C to A to B (ACAB). Any other sequence of points starting at A and ending with B would force the path to trace back over itself.

There are two ways to get a path AB: along the straight line connecting the points, or along the arc.

For the sequence ACAB, let's look at ACA first. There are two ways to trace this sequence of points: along the line and then the arc, or along the arc and then the line. We know that there are then two ways to get from A to B, so we have 2×2=42 \times 2 = 42×2=4 paths tracing the points ACAB.

This gives us 2+4=62 + 4 = 62+4=6 ways to get from A to B without backtracking. There are also 666 ways to get from A to C without backtracking, so we have 6+6=126 + 6 = 126+6=12 ways to get from A to either B or C. We can then extend each of these paths to get to point D.

Using the same strategy found in the introduction, we can simplify the diagram by getting rid of the arc from B to C and start by finding the number of paths that go from A to B without backtracking:

Starting at A, we can go to B, E, or C next. If we choose B, that completes one path (AB). If we choose E, we can go to B or C next, where choosing B completes another path (AEB), and choosing C forces us to go to A and then B, which completes another path (AECAB). Continuing with this reasoning, we can organize the different possible paths with the following tree diagram:

There are therefore 555 different paths that go from A to B without backtracking (AB, AEB, AECAB, ACEAB, and ACEB). By symmetry, there are also 555 different paths that go from A to C without backtracking (AC, AEC, AEBAC, ABEAC, and ABEC). Therefore, there are 5+5=105 + 5 = \boxed{10}5+5=10​ different paths that go from A to D without backtracking.

solution of:

def find_all_paths(start_node, end_node, edge_list):
    """
    finds all paths from start_node to end_node in a graph described with edge_list
    """
    sol_set = set()
    def inner(cur_node, cur_path, cur_edge_list):
        tmp_path = cur_path+[cur_node]
        if cur_node == end_node:
            sol_set.add(tuple(tmp_path))
        else:
            for (edge_num, tmp_edge) in enumerate(cur_edge_list):
                if cur_node in tmp_edge:
                    tmp_edge_list = cur_edge_list.copy()
                    del tmp_edge_list[edge_num]
                    new_node = list(tmp_edge-{cur_node})[0]
                    inner(new_node, tmp_path, tmp_edge_list)
    inner(start_node, [], edge_list)
    return sol_set

def path_to_str(in_edge_list):
    """
    converts a list of nodes into a string
    """
    return ' -> '.join(in_edge_list)

EDGE_LIST = [{'A', 'B'}, {'A', 'C'}, {'B', 'D'}, {'D', 'C'},
             {'B', 'E'}, {'E', 'C'}, {'A', 'E'}]

SOL_SET = find_all_paths('A', 'D', EDGE_LIST)
print(f"number of paths: {len(SOL_SET)}")
for _ in SOL_SET:
    print(path_to_str(_))

Every path to D must use either BD or CD, not both. To consider each case, we delete from the graph the other path. Due to graph's symmetry, we only need to consider one case, for instance, the ones using BD. These ones can only go either through AB and EB, not both, and again we consider each case deleting the other cases from the graph. This way, we reduce our graph to one simpler enough to count easily all the hypotheses.

  • For ones terminating in the sequence ABD, we have 3 hypotheses: ABD, ACEABD and AECABD.
  • For ones terminating in the sequence EBD, we have 2 hypotheses: AEBD and ACEBD.

Adding the numbers and duplicating, we get 10\boxed{10}10​.

We can't use the line BC or else we will have over-tracing.

By symmetry we consider all the ways we can get to B, then we will multiply by 2

We will denote the number of ways to get to a point P as #P.

#B = #A + #E (using only A, C, E or else we would trace-over)

#A = | { (ACEA), (A), (AECA) } | = 3, #B = | { (AE), (ACE) } | = 2.

Thus #B = 5.

By symmetry #D = 10
