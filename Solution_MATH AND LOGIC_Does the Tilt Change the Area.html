Just a thought:
Since the squares are skewed, we should skew the circles.
We get a parallelogram (with same base and height of the square's) and an ellipse.
Mark side | height of square | parallelogram: x.
Area of square | parallelogram = x^2.
Area of circle = (pi / 4) * x^2.
Skewed angle alpha gives us the ellipse's axises(...): Major axis = x / sin(alpha),
Minor axis = x * sin(alpha).
Area of ellipse = pi * (Major axis / 2) * (Minor axis / 2) =
(pi / 4) * (x / sin(alpha)) * (x * sin(alpha)) =
(pi / 4) * x^2 == Area of circle.
All areas keep their ratios to each other (circle / square == ellipse / parallelogram, circle == ellipse, square == parallelogram).
So, if circles were skewed to ellipses, they would fit without overlapping...

First, take the rectangle. In this rectangle, if each circle has radius r, then total height would be 3(2r) = 6r. Given that the second parallelogram has same height, the height of parallelogram is also 6r.

If the 3 circles fit in parallelogram with no overlapping, then the length of the common axis passing through the centers of all circles would have length 6r.

This means that the height and the non-perpendicular axis passing through the centers has same length, which can't be possible as in the parallelogram, we can construct a legal right-angled triangle with the common axis as hypotenuse and height has one of the legs.

So, by this proof of contradiction, we proved that we can't make 3 circles fit in a parallelogram that has same base and height.
