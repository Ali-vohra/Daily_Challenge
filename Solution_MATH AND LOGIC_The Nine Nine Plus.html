Note that the sum of all of the available tiles is 000, and yet the goal is to make the row sum and the column sum both positive. When they first see this puzzle, some people refuse to even try to solve it. They glance at it and claim that it's impossible... we promise, it's not.

If you start by just filling in the row so that it sums to 1,1,1, how many different ways are there to do that?

After you've gotten a three-tile row to equal 1,1,1, try rearranging that row so that you can also make the three-tile column equal 111 using the center tile of the row and the two tiles you didn't use for the row.

    Is it possible?

    The total sum of the tiles given in the grid is 9+9=189+9=\textcolor{#BA33D6}{\boxed{18}}9+9=18​.

    The sum of first five natural numbers is 5(6)2=15\dfrac{5(6)}{2}=\textcolor{magenta}{\boxed{15}}25(6)​=15​.

    Hence the overlap contains 18−15=3\textcolor{#BA33D6}{\boxed{18}}-\textcolor{magenta}{\boxed{15}}=\textcolor{#20A900}{\boxed{3}}18​−15​=3​.

    Putting 3\textcolor{#20A900}{\boxed{3}}3​ in the overlap, we can complete the puzzle by using (2,4)\textcolor{#69047E}{\boxed{(2,4)}}(2,4)​ and (1,5)\textcolor{#69047E}{\boxed{(1,5)}}(1,5)​ in each others' row or column, we can complete the puzzle.

    Hence,

    It is possible for the row and column sum to be 9\textcolor{#D61F06}{\boxed{9}}9​.

    No. of Solutions

    There are four ways to use one pair(opposite to each other) and two ways for the other.

    Hence,

    There are a total of 4×2=84\times2=\textcolor{#D61F06}{\boxed{8}}4×2=8​ solutions.

    The sum of tiles to distribute is 1+2+3+4+5=151+2+3+4+5=151+2+3+4+5=15. If CCC is the central tile, H1H_1H1​ and H2H_2H2​, the extreme horizontals, and V1V_1V1​ and V2V_2V2​, the vertical ones:

    {C+H1+H2+V1+V2=15C+H1+H2=9C+V1+V2=9\begin{cases}C+H_1+H_2+V_1+V_2=15\\C+H_1+H_2=9\\C+V_1+V_2=9\\ \end{cases}⎩⎪⎨⎪⎧​C+H1​+H2​+V1​+V2​=15C+H1​+H2​=9C+V1​+V2​=9​

    But

    {C=(C+H1+H2)+(C+V1+V2)−(C+H1+H2+V1+V2)H1+H2=(C+H1+H2)−CV1+V2=(C+V1+V2)−C\begin{cases}C=(C+H_1+H_2)+(C+V_1+V_2)-(C+H_1+H_2+V_1+V_2)\\H_1+H_2=(C+H_1+H_2)-C\\V_1+V_2=(C+V_1+V_2)-C\\\end{cases}⎩⎪⎨⎪⎧​C=(C+H1​+H2​)+(C+V1​+V2​)−(C+H1​+H2​+V1​+V2​)H1​+H2​=(C+H1​+H2​)−CV1​+V2​=(C+V1​+V2​)−C​

    Substituting values:

    {C=9+9−15=3H1+H2=9−3=6V1+V2=9−3=6\begin{cases}C=9+9-15=3\\H_1+H_2=9-3=6\\V_1+V_2=9-3=6\\\end{cases}⎩⎪⎨⎪⎧​C=9+9−15=3H1​+H2​=9−3=6V1​+V2​=9−3=6​

    To exist a solution, the sum of the vertical extremes must be 6, just like the sum of the horizontal extremes. But, we also know that:

    V1+V2=(C+H1+H2+V1+V2)−(C+H1+H2)=15−9=6V_1+V_2=(C+H_1+H_2+V_1+V_2)-(C+H_1+H_2)=15-9=6V1​+V2​=(C+H1​+H2​+V1​+V2​)−(C+H1​+H2​)=15−9=6

    So, if all the tiles sums to 15, the problem has a solution if there is a tile 3, and we are able to find a pair in the remainder set summing to 6. That implies automatically the validity of the other pair. That's the case of the present problem where there is a tile 3, and the pairs (1,5), and (2,4) sums both to 6. But we could also play with variations in the tiles to put. Instead of {1,2,3,4,5}, we could have:

    −1 1 3 5 7-1\,1\,3\,5\,7−11357

    If all the tiles are different, the number of different graphical dispositions for the solution is 8 (choose an horizontal pair of extremes, rotate it four times to get different dispositions... each rotation leaves fixed places for the other pair of extremes, which supports two type of placements). The following variations do not support has many solutions:

    −1 3 3 3 7-1\,3\,3\,3\,7−13337 1 1 3 5 51\,1\,3\,5\,511355

    The next step in generalization would be to offer sets of tiles with different sums... that would imply (if we keep the same goals for the vertical and horizontal sums) different central pieces and sums for the extremes...

    Ignore the "9" part for now.

    If you want both of them to have the same total, you put the greatest and least values in one so that they aren't in the other. Then you put the next greatest and least numbers in the other. Those will be equal.

    Now you're left with the central value, which goes in the center so that both share it.

    THEN you can work out what the sums actually are, and they are indeed 9:

    • 5+1+3=9
    • 4+2+3=9

    Since they each add up to 9, it's possible.

    """
animation for:
    https://brilliant.org/daily-problems/grid-fill-nine-cross/

import itertools
import matplotlib.pyplot as plt
import matplotlib.patches as patches

def single_frame(in_data, frame_num):
    """
    draws a single frame
    """
    pos_dict = {0:(-1, 0), 1:(0, 0), 2:(1, 0), 3:(0, 1), 4:(0, -1)}
    cur_fontsize = 35
    title_fontsize = 45
    cur_fig, cur_ax = plt.subplots()
    cur_fig.subplots_adjust(top=0.8)
    cur_ax.set_aspect('equal')
    plt.axis('off')
    x_min = -1.5
    x_max = 2
    y_min = -1.5
    y_max = 2
    cur_ax.plot(x_min, y_min, color=[1, 1, 1])
    cur_ax.plot(x_min, y_max, color=[1, 1, 1])
    cur_ax.plot(x_max, y_min, color=[1, 1, 1])
    cur_ax.plot(x_max, y_max, color=[1, 1, 1])
    grid_color = [0.6, 0.6, 0.6]
    grid_line_style = ':'
    d_grid = 0.5
    cur_ax.add_patch(patches.Rectangle((-d_grid, -1-d_grid),
                                       2*d_grid,
                                       2+2*d_grid,
                                       edgecolor=grid_color+[1],
                                       facecolor=[1, 1, 1]+[0],
                                       linewidth=3,
                                       linestyle=grid_line_style))

    cur_ax.add_patch(patches.Rectangle((-1-d_grid, -d_grid),
                                       2+2*d_grid,
                                       2*d_grid,
                                       edgecolor=grid_color+[1],
                                       facecolor=[1, 1, 1]+[0],
                                       linewidth=3,
                                       linestyle=grid_line_style))
    for (cur_ind, cur_num) in enumerate(in_data):
        cur_pos = pos_dict[cur_ind]
        plt.text(cur_pos[0], cur_pos[1], str(cur_num),
                 horizontalalignment='center',
                 verticalalignment='center',
                 multialignment='center',
                 fontsize=cur_fontsize)
    def proc_sum_str(in_sum_val, target_val, str_pos):
        if in_sum_val == target_val:
            cur_str = str(in_sum_val)
            cur_col = [0, 1, 0]
        else:
            cur_str = f"{str(in_sum_val)} ≠ {str(target_val)}"
            cur_col = [1, 0, 0]
        plt.text(str_pos[0], str_pos[1], cur_str,
                 horizontalalignment='center',
                 verticalalignment='center',
                 color=cur_col,
                 fontsize=cur_fontsize)
    sum_h = in_data[0]+in_data[1]+in_data[2]
    sum_v = in_data[3]+in_data[1]+in_data[4]
    target_h = 9
    target_v = target_h
    proc_sum_str(sum_h, target_h, (2.8, 0))
    proc_sum_str(sum_v, target_v, (0, -2))
    if sum_h == target_h and sum_v == target_v:
        plt.title("Correct!", fontsize=title_fontsize)
    file_name = "{:04d}.png".format(frame_num)
    plt.savefig(file_name, dpi=80)
    plt.close(cur_fig)


for (f_num, cur_data) in enumerate(itertools.permutations(range(1, 6))):
    single_frame(cur_data, f_num)
