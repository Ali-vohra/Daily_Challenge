# Whose tiles have the lowest product?
# 
# Sabina = person 0
# Ebrahim = person 1
# Deepa = person 2
#
# The person with the lowest product will be the one holding the zero tile.

def tiles(n,per):
    # returns a string of the tiles given to a person, 
    # e.g. if tiles (n,0) returns "146", person 0 has tiles 1,4, and 6.
    # (no sorting required as the array is already sorted)
    retstring=""
    for loop in range(7):
        retstring = retstring + (str(loop) if n[loop]==per else "")
    return retstring

def sum(n,per):
    # returns a string of the digits of the sum of all tiles
    # belonging to person 'per' sorted numerically,
    # e.g. if the sum of person 0's tiles is 20 then sum(n,0) will return "02"
    # The sorting allows easy comparison of tiles and sums.
    retval=0
    for loop in range(7):
        retval = retval + (loop if n[loop]==per else 0)
    return "".join(sorted(str(retval)))
    #convert into string and sort (the join forces string return)

def product(n,per):
    # returns a string of the digits of the product of all tiles 
    # belonging to person 'per' sorted numerically,
    # e.g. if the product of person 0's tiles is 20 then product(n,0) will return "02"
    # The sorting allows easy comparison of tiles and products.
    retval=1
    for loop in range(7):
        retval = retval * (loop if n[loop]==per else 1)
    return "".join(sorted(str(retval)))
    #convert into string and sort (the join forces string return)    

n=[0,1,2,3,4,5,6] #initilaise list so that we can use loops. The values will be overwritten. 
# n[x] contains the location (person number) of tile x

for n[0] in range(3):

My reasoning:

Each of Sabina, Ebrahim, and Deepa must have at least one tile

Ebrahim and Deepa explicitly state that they have tiles. If Sabina were to have no tiles, both the sum and product of the (no tiles) would be 0, and there is only one 0.

Sabina cannot have the 0 tile

If Sabina's tiles include the 0, the product of her tiles is also 0, an invalid result.

Sabina cannot have only one or two tiles

There are 7 tiles to distribute, and the largest two are 6 and 5. Summing or multiplying 6 and 5 results in a two-digit answer, leaving one tile unused.

Sabina cannot have four or more tiles

The smallest non-zero-containing set of four tiles is 1, 2, 3, and 4. The sum of these is 10, and the product of them is 24. This would require eight tiles.

Sabina has 3 tiles by process of elimination.

Ebrahim and Deepa have two tiles each

The only three-digit product of the available tiles is 6Ã—5Ã—4=120, and the sum 6+5+4=15 is an invalid result both for using too many digits and for reusing digits.

Ebrahim must have the 1 tile

The largest sum of three tiles is 15, so all valid two-digit sums include the digit 1.

Sabina must have the at least one of the tiles 5 and 6

For the sum of three tiles to be 10 or greater, at least one tile must be greater than 4, as 4+3+2=9.

Sabina has the 5 tile but not the 6 tile, Deepa has the 0 tile

No product of 6 and two other tiles which are not 5 contains a 0. The only sum which contains a 0 is 10, and without using the 1 tile, the only tile which work are 5+3+2=10, but the product 5Ã—3Ã—2=30 invalidates this combination. Therefore If Deepa has the 0, Sabina has the 5. 6Ã—5Ã—2=60, an invalid answer which is both the lowest set of three tiles available to Sabina with both 6 and 5, and the highest combination of two tiles containing a 0.

We've answered the question, as Deepa's product must be 0.

Finishing up

At this point, only two possible sets of three tiles remain for Sabina. These are 5, 4, 3 and 5, 4, 2.

5+4+2=11. This answer won't work.

5+4+3=12, 5Ã—4Ã—3=60. This uses all seven tiles and is the completed puzzle.

The person who has the smallest product of tiles will be the one who contains the 0 tile because anything multiplied by 0 is 0.

Sabina

Sabina cannot have the 0 tile because then their product would be 0 and then to fulfil Deepa's condition another 0 would be required.

Ebrahim

If Ebrahim has the 0 tile then they must also have the 1 tile because it is impossible to create a sum of a multiple of 10 (using the tiles we have) excluding 10 itself.

For Ebrahim to have the 1 and 0 tiles then that must mean that Sabina's tiles sum to 10 as well.

These are the ways in which Sabina can have their tiles sum to 10 (Without using the 1 and 0 tiles which are already taken):

6 + 4

5 + 3 + 2

If Ebrahim has the 6 and 4 tiles then Deepa will have to have the 2 and 4 tiles which cannot happen because 4 is taken.

If Ebrahim has the 5, 3 and 2 tiles then Deepa will have to have the 3 and 0 tiles which are both already taken.

Deepa

By process of elimination Deepa will have the 0 tile and therefore the smallest product.

Solving the puzzle itself (Not required to complete the challenge):

We know that Deepa has the 0 tile so Sabina's product must be a multiple of 10.

Ways Sabina can have a multiple of 10:

6 * 5 (Won't satisfy Ebrahim's condition because 6+5 = 11 and would require two 1 tiles.)

6 * 5 * 4 (Won't satisfy Ebrahim's condition because 6+5+4 = 15 and the 5 tile is taken.)

6 * 5 * 2 (Won't satisfy Deepa's condition because 652 = 60 and the 6 tile is taken.)

5 * 4 (Won't satisfy Ebrahim's condition because 5+4 = 9 and there is no 9 tile.)

5 * 4 * 3 (Satisfies all conditions!)

There is no need to find any more ways Sabina can have a multiple of 10.

5 * 4 * 3 = 60 so Deepa will have the 6 tile.

5 + 4 + 3 = 12 so Ebrahim will have the 1 and 2 tiles.
