A close look at the curious graph will show that the six blue dots across the center represent the six faces of the blue die { 5, 5, 5, 5, 5, 5 }.

Similarly, the six red dots represent the six faces of the red die { 3, 3, 3, 3, 9, 9 }, and so on for the orange die { 1, 1, 7, 7, 7, 7 }.

The six x's show that { 0, 0, 2, 2, 8, 8 } loses against all of the previous die because the four faces { 0, 0, 2, 2 } lose to the previous dice. In this sense, { 0, 0, 2, 2, 8, 10 } is another version of an all-losing die.

Again, similar considerations would show that { 0, 2, 8, 10, 10, 10 } is an all-win die.

The { 2, 2, 2, 8, 8, 8 } die will win against orange, lose against red and tie blue.

The graph also shows why { 4, 4, 4, 4, 4, 4 } is, in a sense, a transitive die because it loses to the blue die but wins against the red and orange dice. This die does not sum to 30.

Note that { 6, 6, 6, 6, 6, 6 } is an all-win die..

A die like { 2, 2, 2, 6, 6, 6 } that ties the blue die can still lose to the red and orange dice and so is not transitive.

A die like { 0, 0, 0, 0, 0, 30 } would sum to 30 and still lose to the blue, red and orange dice, so its not transitive.

There are many other transitive die that either lose or win against the blue die, which happens to be symmetrically in the center of this problem.

This is a solution for @Mahdi Raza 's followup questions.

How many solutions are there?

Infinitely many. Consider the die [3, 3, 3, 4, 4, 10]. This loses on average to blue, but wins on average compared to red (12 draws, 14 wins, 10 losses). Since the 10 is an automatic win for the die, we can add pips to that without changing its winrate. So we can have the die [3, 3, 3, 4, 4, 10 + n], where n is a positive integer, as a die that wins against red and loses against blue, which implies there are infinitely many solutions.

What's the arrangement of the die that has the smallest number of pips?

[0, 4, 4, 4, 4, 4], which loses to blue and wins against red (0 draws, 20 wins, 16 losses). This was done via computer program. The gist of the code was to generate every arrangement of 6 numbers, all less than 11, such that the arrangement is in sorted order (this keeps duplicates out, which helps efficiency and makes it easier to code), then see if that arrangement meets the conditions. This code isn't as fast as it could be; once a smaller pip count is found, it still tries to generate numbers with a necessarily larger pip count. I'll revise this in the future, but this works for now. There might be other dice with a pip count of 20 that also work.

The reason all sides need to be less than 11 is because as previously mentioned, 10 pips is an automatic win. Any side with more than 10 pips can have pips removed without changing the win rate; to find the minimum, all sides need to have fewer than 11 pips.

Line 47 is an efficiency thing; once a die has 3 sides with 10 pips, it can't satisfy the outlined property, as it will always have 1/2 chance of winning; the worst it can do is draw.

#include <iostream>

int compare(int* d1, int* d2) {
    int d1wins = 0;
    int d2wins = 0;

    for (int i = 0; i < 6; i++) {
        for (int j = 0; j < 6; j++) {
            d1wins += d1[i] > d2[j]; // Little bit of code golfing; booleans (the RHS) have a value of 0 or 1,
            d2wins += d2[j] > d1[i]; // so adding them in this way counts how many times the RHS is true.
                                     // Looping over all the values, combined with this adding, counts how many wins.
        }
    }

    if (d1wins > d2wins) return 10; // d1 beats d2
    else if (d1wins == d2wins) return 0; // d1 draws with d2
    else return 1; // d1 loses to d2
    // The reason the numbers are what they are is because it gives a nice representation of a win-loss record;
    // if the die has 2 wins and 1 loss, then its record is 2-1, which is used in the code as 21. Against the
    // given dice, we want both of those digits to be nonzero.
}

void generateNextDie(int* die, int index, int** dice, int& minPipValue, int* bestDie) {
    // We place a number at index. The numbers at index + 1 and onward are unaffectable.
    if (index == -1) {
        // Neato, we've constructed a die!
        int record = 0;
        for (int i = 0; i < 3; i++) record += compare(die, dice[i]);

        if (record / 10 > 0 && record % 10 > 0) {
            int sum = 0; 
            for (int i = 0; i < 6; i++) sum += die[i];

            if (sum < minPipValue) {
                for (int i = 0; i < 6; i++) bestDie[i] = die[i];
                minPipValue = sum;
            }
        }

    } else if (index != 2 || die[3] != 10) {
        for (int i = 0; i <= die[index + 1]; i++) {
            die[index] = i;
            generateNextDie(die, index - 1, dice, minPipValue, bestDie);
        }
    }
}

void findSmallestWorkingDice(int* d1, int* d2, int* d3) {

    int size = 6;
    int* testDie = new int[size];
    for (int i = 0; i < size; i++) testDie[i] = 0;
    testDie[4] = 1; testDie[5] = 1;

    int** allDice = new int*[3];
    allDice[0] = d1;
    allDice[1] = d2;
    allDice[2] = d3;

    int* bestDie = new int[6];
    int pips = 30;

    while(testDie[size - 1] < 11) {
        generateNextDie(testDie, size - 2, allDice, pips, bestDie);
        testDie[size - 1]++;
    }

    std::cout << "The die with the smallest pip count is";
    print(bestDie, 6);
    std::cout << ", with a pip count of " << pips << "." << std::endl;

    delete [] testDie;
    delete [] bestDie;
    delete [] allDice;
}

    int* redDie = new int[6];
    for (int i = 0; i < 4; i++) redDie[i] = 3;

    wins = 0
    for a in die1:
        for b in die2:
            if a > b: wins += 1

A relation is transitive if it has the property that follows: if the relation holds between aaa and bbb and also between bbb and c,c,c, then it is guaranteed to hold between aaa and c.c.c. For example, equality (=) is transitive, so if we know that x=y,x=y,x=y, and we also know that y=7y=7y=7, we can conclude that x=7x=7x=7.
