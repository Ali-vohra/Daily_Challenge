Math and Logic

Add to Twelve

Using each of the digits 111 through 888 at most once each, how many ways can we get a sum of 8?8?8? We could try a guess-and-check approach, but we might end up missing a combination or double-counting another. Keep reading to see a systematic counting approach, or jump ahead to today's challenge if you feel ready.

Let's think about how we can use the ordering of the digits to our advantage, and break our problem down into smaller, more easily digestible sub-problems. Doing that will give us some structure and make it less likely that we miscount. We'll start by using the larger digits, and then work our way down to the smaller ones.

Our first sub-problem will be, how many ways can we include the digit 8\mathit{8}8 in the sum? This one has a quick answer: there is only one way, i.e. by using 888 with no other digits. There is also only one sum that includes 7,7,7, specifically 7+1.7 + 1.7+1.

The simplest sum including 666 is 6+2,6 + 2,6+2, which leads us to a smaller sub-problem: how many ways can we get a sum of 222 using digits we have? Digits can't be used more than once, so 6+1+16 + 1 + 16+1+1 isn't allowed. This means we can only get a sum of 222 from 222 itself, so the one way to get a sum of 888 while including 666 is 6+2.6 + 2.6+2.

It might seem like we were making things too complicated with the last case, but that reasoning will come in handy here. To get a sum including 5,5,5, we can choose 5+3,5 + 3,5+3, but we should ask ourselves, "Is there another way to get a sum of 333 from cards other than 5?5?5?"

There are two ways to get a sum of 333 from the available digits: just 333 by itself, or 2+1.2 + 1.2+1. So, there are two ways to make a sum of 888 that includes 5:5:5: 5+3,5+2+1. 5 + 3, \quad 5 + 2 + 1. 5+3,5+2+1. How about a sum including 4?4?4? While 8=4+4,8 = 4 + 4,8=4+4, we can't use 444 twice. Let's see how many ways we can get a sum of 444 from our digits: 3+13 + 13+1 is allowed, but 2+22 + 22+2 is not. The one way to get a sum including 444 is 4+3+1.4 + 3 + 1.4+3+1.

For a sum including 3,3,3, we might think 3+5=8,3 + 5 = 8,3+5=8, but we already counted 5+3.5 + 3.5+3. Breaking 555 into 4+14 + 14+1 to get 3+4+13 + 4 + 13+4+1 gives us another double-counting â€” this time double-counting of 4+3+1.4 + 3 + 1.4+3+1. To avoid double-counting, we can consider only smaller digits for each addition. A sum of 555 from the digits smaller than 333 is impossible, so we've already found all sums that include 3.3.3. We can use similar reasoning to conclude that we've already found all sums that include 222 and that include 1,1,1, too.

So, we have 1+1+1+2+1=6 1 + 1 + 1 + 2 + 1 = \mathbf{6} 1+1+1+2+1=6 ways to get a sum of 888 by using each digit at most once. By organizing our approach in this way, we made sure we didn't miss any combinations or double-count any.
