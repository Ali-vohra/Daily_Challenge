Where Does the Ball Land?

If you wanted to find out which pocket the billiard ball falls in, you could try making a drawing, but that approach might not work very well on today's challenge. Keep reading to see another strategy.

We'll assume geometric purity here: the ball has no mass, the ball and hole are considered single points, and the ball travels straight. Which hole will the ball ultimately enter in the animation above (or will it bounce around forever)?

To be precise, we want to consider the situation below: the ball starts in the bottom left corner and is shot at a perfect 45âˆ˜45^\circ45âˆ˜ angle.

To solve this, let's think about the table's dimensions and the characteristics of the 45âˆ˜45^\circ45âˆ˜ angle. Because the angle measures 45âˆ˜,45^\circ,45âˆ˜, the ball will always move the same distance vertically as it moves horizontally. So, the first bounce will occur 676767 inches from the left on the top rail.

When does it sink in a hole? Only when the ball exactly strikes one of the corners, which means that the horizontal distance needs to be a multiple of 100100100 and the vertical distance needs to be a multiple of 676767 at the same time. Since the vertical and horizontal distances are always equal, the first time this will happen is when the ball has moved 670067006700 inches both vertically and horizontally. So we know if the ball keeps moving, it will sink in one of the holesâ€¦ eventually. But which one?

When tracked along the top of the table, the ball moves the full length of the table â€” 100100100 inches â€” so after 670067006700 inches of horizontal movement, it will have traversed the table horizontally 676767 times: 343434 times going left to right, and 333333 times going right to left. Therefore, it will end its movement, falling into a hole on the right-hand side. We also know that the ball will traverse the table vertically 100100100 times: 505050 times going bottom to top, and 505050 times going top to bottom. Therefore, it will end its movement at the bottom of the table. Combining these two constraints, we can conclude that the ball, after a great many bounces, will fall into the bottom right corner pocket.

Can the same reasoning be applied to today's challenge?

Direction coding

We can use numbers to represent four directions:

What are the benefits of such coding?

  1. Easy to program

  2. We can find:

    • If the ball hits the upper edge, the direction of the ball at this time can only be 0 or 1 (2 and 3 directions are going down, and it is impossible to touch the upper edge when going down), and it needs to rebound, that is, direction 0 becomes Direction 2, direction 1 becomes direction 3, so we only need to add the direction value + 2.

    • In the same way, we can get that if the ball hits the bottom edge, we only need to add the direction value to -2.

    • If the ball hits the left edge, we only need to add the direction value +1.

    • If the ball hits the right edge, we only need to add the direction value -1.

So we only need to set the amount that needs to be changed on the edge of the table, and then we can easily calculate the new direction.

Endless loop

How to judge whether the ball will loop indefinitely?

We can store the state of each step of the ball (that is, the current coordinates and direction) into the array. If we find that the current state has appeared before, it means that we have returned to the previous state. If we continue to walk, we will fall into Into an endless loop.

This may be a bit difficult to understand. Take our daily example, if you keep going forward and want to reach the end, but you discovered that you have been to this place before, and if you keep going in the original direction, you will never end.

The format for storing the current state here is a five-digit number: the first digit is the direction, the second and third digits are the x-coordinates, and the fourth and fifth digits are the y-coordinates. For example, 10101\color{#D61F06}1\color{#3D99F6}01\color{#20A900}0110101 represents the direction1\color{#D61F06}11, and the x coordinate is 01\color{#3D99F6}01 01, the y coordinate is 01\color{#20A900}0101.

As long as we keep updating the position of the ball until it falls into the bag, we can simulate every step of the ball.

Yes, you can solve this for a general angle. For that, each time the ball bounces against an edge, reflect the pool table from that edge and track the trajectory of the ball in the reflection. Then you're looking at a straight line and landing in a hole is equivalent to the line passing through one of the points in an infinite lattice. If the ratio of the sidelengths of the table is rational (which is the case here since 6/14 is rational), you can deduce that this can only happen if the tangent of the angle is rational. This is not true for 30 degrees or 60 degrees. This means that the ball will bounce around indefinitely in both cases.

UNDERSTANDING THE QUESTION:

  • The pool table is 14 m x 6 m.

  • The ball is hit a a 45Â° from the top left.

  • With 45Â° angles, the horizontal and vertical distances are equal.

  • The ball will never stop UNLESS it is potted in any 1 of the 4 pockets.

  • It will always travel in 45Â° angles.

FROM THE INTRO:

"When does it sink in a hole? Only when the ball exactly strikes one of the corners, which means that the horizontal distance needs to be a multiple of 100 and the vertical distance needs to be a multiple of 67 at the same time. Since the vertical and horizontal distances are always equal, the first time this will happen is when the ball has moved 6700 inches both vertically and horizontally. So we know if the ball keeps moving, it will sink in one of the holesâ€¦ eventually. But which one?"

STEP 1: Find the LCM.

  • The LCM of 14 and 6 is 42.

  • This means that once the ball has moved 42 m both horizontally and vertically, it will be potted in one of the pockets.

FROM THE INTRO:

"When tracked along the top of the table, the ball moves the full length of the table â€” 100 inches â€” so after 6700 inches of horizontal movement, it will have traversed the table horizontally 67 times: 34 times going left to right, and 33 times going right to left. Therefore, it will end its movement, falling into a hole on the right-hand side. We also know that the ball will traverse the table vertically 100 times: 50 times going bottom to top, and 50 times going top to bottom. Therefore, it will end its movement at the bottom of the table. Combining these two constraints, we can conclude that the ball, after a great many bounces, will fall into the bottom right corner pocket."

STEP 2: Determining the Pocket (PART 1)

How many times will the ball move across the full length of the pool table (i.e. horizontally)?

  • The table is 14 m long, so after 42 m worth of horizontal distance, the ball will have traversed the table 3 times.

How many times will the ball move across the full width of the pool table (i.e. vertically)?

  • The table is 6 m wide, so after 42 m worth of vertical distance, the ball will have traversed the table 7 times.

STEP 2: Determining the Pocket (PART 2)

What does it mean that the ball has traversed the table 3 times in length and 7 times in width?

Length

  • The ball would have moved in this manner:

Right â€“ Left â€“ Right.

  • In other words, after traversing the table 3 times in length, the ball will end up on the right side of the table.

Width

  • The ball would have moved in this manner:

Down â€“ Up â€“ Down â€“ Up â€“ Down â€“ Up â€“ Down.

  • In other words, after traversing the table 7 times in width, the ball will end up on the lower half of the table.

Combining these 2 constraints (see below), we can determine the exact pocket the ball will fall into.

  1. After traversing the table 3 times in length, the ball will end up on the right side of the table.

  2. After traversing the table 7 times in width, the ball will end up on the lower half of the table.

  • The ball will end up in the bottom right pocket.

FINAL ANSWER: It falls in the bottom right pocket.

    int width;
    int height;
    int xvelocity;
    int yvelocity;
    int x = 0;

    public BallPuzzle(int width, int height, int xvelocity, int yvelocity) {
        this.width = width;
        this.height = height;
        this.xvelocity = xvelocity;
        this.yvelocity = yvelocity;
    }

    public String solve() {
        boolean inPocket = false;
        String result;
        while (!inPocket) {
            x = x + xvelocity;
            y = y + yvelocity;
            if ((x == 0 || x == width) && (y == 0 || y == height)) {
                inPocket = true;
                result = "Potted in ";
                if (yvelocity > 0) {
                    result = result + "Bottom ";
                } else {
                    result = result + "Top ";
                }
                if (xvelocity > 0) {
                    result = result + "Right ";
                } else {
                    result = result = "Left ";
                }
                return result + "Pocket";

            }
            if (x == 0 || x == width) {
                xvelocity = -xvelocity;
            }
            if (y == 0 || y == height) {
                yvelocity = -yvelocity;
            }
        }
        return "Cannot solve";
    }

    public static void main(String[] args) {
        BallPuzzle ballPuzzle = new BallPuzzle(14, 6, 1, 1);
