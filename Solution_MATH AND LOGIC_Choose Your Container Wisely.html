The least amount of water you can accurately measure at any given time can be found by finding the greatest common factor of the two containers.

  • gcf(9, 12) = 3
  • gcf(10, 12) = 2
  • gcf(12, 15) = 3
  • gcf(12, 16) = 4

The 9 and 15 liter containers can measure out liters in multiples of 3 (3, 6, 9, 12, 15, 18, etc.)

The 10 and 12 liter containers can measure out liters in multiples of 2 (2, 4, 6, 8, 10, 12, 14, etc.)

The 12 and 16 liter containers can measure out liters in multiples of 4 (4, 8, 12, 16, etc)

Thus, you will get more exact measurements out of the 10 and 12 liter containers.

class Container {
  constructor(max) {
    this.max=max;
    this.value=0;
  }
  available() { return this.max-this.value; }
  fill() {
    this.value=this.max;
    return this.value;
  }
  isFull() { return this.value==this.max; }
  isNotFull() { return this.value!=this.max; }
  empty() {
    this.value=0;
    return this.value;
  }
  isEmpty() { return this.value==0; }
  isNotEmpty() { return this.value!=0; }
  add(amount) {
    var retVal=0;
    if(this.value+amount>this.max) {
      retVal=this.value+amount-this.max;
      this.value=this.max;
    } else {
      this.value+=amount;
    }
    return retVal;
  }
  pour(amount) {
    this.value-=amount;
    return this.value;
  }
}

(defun get_volumes (vol_a vol_b)
    ;returns the set of volumes possible to generate with two buckets of given volumes
    (let
        (
            (sol_set ()) 
        )
        (labels
            (
                (inner (cur_vol)
                    (if (and (and (>= cur_vol 0)
                              (or (<= cur_vol vol_a) (<= cur_vol vol_b)))
                         (not (member cur_vol sol_set)))
                        (progn
                            (setq sol_set (adjoin cur_vol sol_set))
                            (inner (- vol_a (- vol_b cur_vol)))
                            (inner (- vol_b (- vol_a cur_vol)))
                        )
                    )
                )
            )
            (inner 0)
            (inner vol_a)
            (inner vol_b)
            sol_set
        )
    )
)

Which container lets you measure out the largest number of different amounts of water with the 12L container?

The question is not well put. After all, any of the candidate solutions allows to measure an infinite enumerable number of different amounts of water, and the cardinals for these sets are the same, â„µ0\aleph_0â„µ0â€‹. There are as many even integers numbers as integers themselves, for instance. Instead, what is probably asked is which solution has the best resolution. Which one allows to measure smaller quantities?

The solution is 10, because the greatest common divisor between 12 and 10 is the smallest in comparison with the other solutions:

gcd(12,9)=3gcd(12,10)=2gcd(12,15)=3gcd(12,16)=4\begin{aligned}gcd(12,9)&=3\\ gcd(12,10)&=\boxed{2}\\ gcd(12,15)&=3\\ gcd(12,16)&=4\\ \end{aligned}gcd(12,9)gcd(12,10)gcd(12,15)gcd(12,16)â€‹=3=2â€‹=3=4â€‹

PS. (2020/04/01): It appears my statement about how many quantities is possible to measure, phased out people. My fault! I should have chosen an interpretation that preserved more the original sense of the problem ("how many") than focus on the word "measure" ("how small can we measure"). For instance, "counting the number of different quantities that can appear on the containers following the rules of the game, which partner container would had lead to the biggest number"?

Although both interpretations leads to the same answer on this problem, they are not the same and could have easily give different results. For instance, had we the choice of a 21-liter container too, that would have allowed 12 different possible quantities on the containers (or 8 maximum for a single one).   2. You cannot measure more than the combined total of the two containers

The number of possible accurately measurable amounts of water with two buckets of volumes x and y can be determined by adding the two volumes (x+y), dividing by the greatest common factor of the two volumes (GCF(x, y)), and adding 1 to include the possibility that both buckets are empty.

This equation looks like:
