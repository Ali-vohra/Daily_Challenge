Place the Zero

Grid Fill puzzles like the one below are similar to Sudoku puzzles in that they ask you to place numbers in squares in a certain way, but the similarities end there.

If you're familiar with how these work, jump straight to today's problem. If you want to look through some worked examples, though, keep reading.

One of the differences between these puzzles and Sudoku is that a Grid Fill solution can have a number appearing more than once in the same row or column. The goal is to drag number tiles from the bank to squares on the grid so the rows and columns sum to the indicated values.

For example, a solution to the puzzle below will have numbers in the top row with a sum of 3. Note that the middle square of that row is gray and will not accept a number tile, so its effective value is zero. By clicking the right arrow below the puzzle, you can view a demonstration of a puzzle solution complete with the underlying logic. If you need to see a step again, click the left arrow.

This is a first solution. Believe it or not, this solution excludes hypothesis 2 and 6. The reason is on the puzzle symmetry. Because the middle sums are zero, the sums in the corners are opposite to each in the same direction, and because the distribution of numbers available is symmetrical in relation to zero, if we have a solution, we can get another solution by multiplying all numbers by -1, and exchange side columns and rows:

The 0 can not be in left column, as we would then need both 1s to sum us to 2 in that column, which would make it impossible to solve on middle column (you need a 1 or a zero to sum to 0 and there are none left). The 0 can not be in right column either, as would need both -1s to sum to -2 in that column, which would make it impossible to solve on the middle column (you need a -1 or a zero to solve to o and there are none left).

So there are between 0 and 3 places the zero could go (top, middle or bottom of middle column or nowhere at all). I did trial and error to find 0 could fit in any of these slots.

% Solution to Brilliant Daily Challenge from 04-04-2020 "Place the Zero"
% written in MATLAB R2018b

Tiles = [-2,-2,-1,-1,0,1,1,2,2];
Number = length(Tiles);
MatrixSize = sqrt(Number);

ColumnSumDesired = [2,0,-2];
RowSumDesired = [3,0,-3];

Permutations = perms(Tiles);
Columns = Permutations(:,1:MatrixSize:end);
Rows = Permutations(:,1:1:MatrixSize);

ColumnSum = zeros(size(Permutations,1),MatrixSize);
RowSum = zeros(size(Permutations,1),MatrixSize);

for ind = 1:1:MatrixSize
    Columns(:,1+MatrixSize*(ind-1):1:MatrixSize*ind) = Permutations(:,ind:MatrixSize:end);
    Rows(:,1+MatrixSize*(ind-1):1:MatrixSize*ind) = Permutations(:,1+MatrixSize*(ind-1):1:MatrixSize*ind);
    ColumnSum(:,ind) = sum(Columns(:,1+MatrixSize*(ind-1):1:MatrixSize*ind),2);
    RowSum(:,ind) = sum(Rows(:,1+MatrixSize*(ind-1):1:MatrixSize*ind),2);
end

ColumnCheck = ismember(ColumnSum,ColumnSumDesired,'rows');
RowCheck = ismember(RowSum,RowSumDesired,'rows');

ColumnIndices = find(ColumnCheck == 1);
RowIndices = find(RowCheck == 1);
CorrectPossibilities = intersect(ColumnIndices,RowIndices);
CorrectPermutations = unique(Permutations(CorrectPossibilities,:),'rows');
[~,ZeroPositions] = find(CorrectPermutations == 0);
NumberOfZeroPositions = length(unique(ZeroPositions));
fprintf("There are %d different positions possible for zero tile.\n",NumberOfZeroPositions)

bool checkData(const std::vector<int>& solData)
{
    //checks if solData satisfies:
    //https://brilliant.org/daily-problems/2020-04-02/
    bool res = true;
    if(solData.size() == 3)
    {
        if(solData[0]+solData[1]+solData[2] != 3)
        {
            res = false;
        }
    }
    else if(solData.size() == 6)
    {
        if(solData[3]+solData[4]+solData[5] != 0)
        {
            res = false;
        }
    }
    else if(solData.size() == 9)
    {
        if(solData[6]+solData[7]+solData[8] != -3)
        {
            res = false;
        }
        else if(solData[0]+solData[3]+solData[6] != 2)
        {
            res = false;
        }
        else if(solData[1]+solData[4]+solData[7] != 0)
        {
            res = false;
        }
        else if(solData[2]+solData[5]+solData[8] != -2)
        {
            res = false;
        }
    }
    else if(solData.size() > 9)
    {
        throw std::invalid_argument("Wrong input data!");
    }

    return res;
}

template<typename DataType>
std::string toRawString(const DataType& inData)
{
    std::stringstream ss;
    for(typename DataType::const_iterator it = inData.begin(); it != inData.end(); ++it)
    {
        ss<<*it;
    }
    return ss.str();
}

void updateSolutiuonData(const std::list<int>& numberList,
                         bool (*checkDataFun)(const std::vector<int>&),
                         const std::vector<int>& curSolutionData,
                         std::set<std::vector<int> >& solutionSet,
                         std::unordered_set<std::string>& hashData)
{
    const std::string inputHash = toRawString(curSolutionData);
    const auto findIt = hashData.find(inputHash);
    if(findIt == hashData.end())
    {
        hashData.insert(inputHash);
        if(checkDataFun(curSolutionData))
        {
            if(numberList.empty())
            {
                solutionSet.insert(curSolutionData);
            }
            else
            {
                auto tmpSolutionData = curSolutionData;
                auto tmpNumberList = numberList;
                const auto numberListSize = numberList.size();
                for(std::size_t i = 0; i < numberListSize; ++i)
                {
                    int curNum = tmpNumberList.front();
                    tmpNumberList.pop_front();
                    tmpSolutionData.push_back(curNum);
                    updateSolutiuonData(tmpNumberList, checkDataFun, tmpSolutionData, solutionSet, hashData);
                    tmpNumberList.push_back(curNum);
                    tmpSolutionData.pop_back();
                }
            }
        }
    }
}

std::set<std::vector<int> > findSetOfSolutions(const std::list<int>& numberList,
                                               bool (*checkDataFun)(const std::vector<int>&))
{
    std::set<std::vector<int> > solSet;
    std::unordered_set<std::string> hashData;
    updateSolutiuonData(numberList,
                        checkDataFun,
                        std::vector<int>(),
                        solSet,
                        hashData);
    return solSet;
}

void displaySolutionSet(const std::set<std::vector<int> >& solutionSet, const unsigned rowLength)
{
    for(const auto& curSol : solutionSet)
    {
        for(std::size_t i = 0; i < curSol.size(); ++i)
        {
            std::cout<<curSol[i];
            if(i%rowLength == rowLength-1)
            {
                std::cout<<'\n';
            }
            else
            {
                std::cout<<'\t';
            }
        }
        std::cout<<"\n";
    }
}

unsigned countDifferentPositionsOfZero(const std::set<std::vector<int> >& solutionSet)
{
    std::set<unsigned> posSet;
    for(const auto& curSol : solutionSet)
    {
        for(unsigned curPos = 0; curPos < curSol.size(); ++curPos)
        {
            if(curSol[curPos] == 0)
            {
                posSet.insert(curPos);
                break;
            }
        }
    }
    return posSet.size();
}

grid = [0 for i in range(9)]
numbers = [-2, -2, -1, -1, 0, 1, 1, 2, 2]
numbers_pos = set(range(9))
solutions = set()

def evaluate(n):
    if n == 9:
        solutions.add(tuple(grid))
        return solutions
    else:
        for np in list(numbers_pos):
            grid[n] = numbers[np]
            if n == 2:
                if grid[0] + grid[1] + grid[2] != 3:
                    continue
            elif n == 5:
                if grid[3] + grid[4] + grid[5] != 0:
                    continue
            elif n == 8:
                if not (grid[6] + grid[7] + grid[8] == -3
                    and grid[0] + grid[3] + grid[6] == 2
                    and grid[1] + grid[4] + grid[7] == 0
                    and grid[2] + grid[5] + grid[8] == -2):
                    continue
            numbers_pos.remove(np)
            evaluate(n + 1)
            numbers_pos.add(np)
    return solutions

for element in evaluate(0):
    print(element)

It's useful to look at this problem as a parity (odd/even) issue.

Only the first and third rows are odd, so they must contain a single one. (There aren't enough ones for a triple in both rows.)

Similarly, each column must contain a pair of ones if it has any at all.

This allows us to exclude zeros from the outer columns as we don't have enough zeros for both 0+1+1 and 0-1-1, so a two is necessary, plus a pair of ones or twos that cancel.

David Vreken's earlier post illustrates five arrangements that satisfy these parity restraints.

Symmetry allows us to rotate any solution by 180 degrees and multiplying everything by -1 gives us a paired solution.

2,0,1 in the top row with 1,-1,-2 in the right column supports two solutions. -1, 2,-1 in the middle row with 1,-2,-2 in the bottom row OR (because the vertical pairs cancel) 1,-2,-2 in the middle row with -1, 2,-1 in the bottom row.

2, 2,-1 in the top row with -1, 1,-2 in the right column is also a solution. with 2, 0,-2 in the center column. While you can still rotate and multiple by -1, the inverted configuration is identical to the initial configuration.

So there are a total of 5 unique configurations, and they all have a single zero in the middle column.

First observation: there's symmetry in both the tiles and the grid. Flip the sign of all the tiles, and you get exactly the same numbers back (in a different order). Meanwhile, flip all the signs on the totals around the grid, and you get the same arrangement as by rotating it 180 degrees. That means you only need to consider the 0 being in five of the nine spaces since, if you find a solution with the 0 in, say, the middle of the top row, you would know there's a second solution with the 0 in the middle of the bottom row, which you get from the first by rotating the arrangement by 180 degrees and flipping all the signs of the tiles. Similarly, if you can show that there are no solutions with a 0 in the middle of the left column, you know there are no solutions with a 0 in the middle of the right column because the same transformation would turn such a solution into one you've just proved doesn't exist. Only the centre space doesn't have a counterpart.

Second observation: the only way a 0 can be in the left column is if the other two tiles there are both 1, but that causes problems with the top row - taking pairs of the remaining tiles, they can only sum to -4, -3, -2, 0, 1, or 4, and adding 0 or 1 to any of those totals can't make the required total of 3.

Combining those two observations shows that 0 can only be in the middle column. If it's on the top row, then the other two spaces have to be 1 and 2. Without a 0, the bottom row must be -2, -2 and 1. That leaves -1, -1 and 2 in the middle row. The right column has to be 1 or 2, -1 or 2 and -2 or 1 with a total of -2, meaning you must take the lower of each pair. That leaves a 2 in top left. The remaining two numbers in the middle row, -1 and 2, can go either way around, with 1 placed below the -1 and -2 below the 2 to give two possible arrangements with 0 in the middle of the top row. From the first observation, that also gives two arrangements with 0 in the middle of the bottom row.

Finally, for 0 in the centre, the top row has to be -1, 2 and 2 in some order to make 3, and the bottom row has to be -2, -2 and 1 in some order to make -3, leaving 1 and -1 in the middle row. The left column has to make a total of 2 out of -1 or 2, -1 or 1 and 1 or -2, which can only be 2, -1, 1 reading down the column. The same argument gives -1, 1, -2 down the right column (there's that symmetry again), leaving 2 in top middle and -2 in bottom middle, which gives the right total for that column too, and the third place 0 can go.

So that's a little more work than needed (I didn't need to find both solutions with 0 in top-middle) to show not only that there are 3 places the 0 can go, but also that there are 5 solutions in total.
