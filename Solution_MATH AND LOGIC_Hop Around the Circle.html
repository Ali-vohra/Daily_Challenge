Hop Around the Circle

One way to make an eight-pointed star is by starting with a circle. Make eight dots, and then start at the top dot on the circle. If we repeatedly jump ahead 3 dots, we can see where to draw the next segment.

The star is completely drawn now, but we could continue the pattern as long as we want, tracing over the star. Which dot would we land on after drawing the 100th100^\text{th}100th segment? Keep reading to find out, or jump ahead to today's challenge for a different question.

We really don't want to draw 100100100 segments to answer this question. Let's see if there's a pattern we can find instead.

We start at dot 0,0,0, and then skip ahead 333 dots to dot 333 for the first segment. Skipping ahead 333 more takes us to dot 6.6.6. Tracing out the complete star, we visit the dots in this order:

0→3→6→1→4→7→2→5→0… 0 \rightarrow 3 \rightarrow 6 \rightarrow 1 \rightarrow 4 \rightarrow 7 \rightarrow 2 \rightarrow 5 \rightarrow 0 \ldots 0→3→6→1→4→7→2→5→0…

This cycle keeps repeating as long as we continue to trace over the star.

It looks like we have a nice pattern in the first three terms, adding 333 to get the next term each time, but that pattern breaks when we go from 666 to 1.1.1. Adding 333 to 666 gives us 9,9,9, but there isn't any dot numbered 999 on the star. To find which dot it is, we can subtract 888 to get 9−8=1.9 - 8 = 1.9−8=1. That lets us go back to adding 333 again to get dot 4,4,4, again to get dot 7,7,7, but adding 333 again takes us to 10.10.10. We can again subtract 888 to see we're at dot 10−8=2,10 - 8 = 2,10−8=2, which takes us next to dot 5,5,5, and finally dot 8−8=0.8 - 8 = 0.8−8=0.

When we subtracted 8,8,8, whether it was once or twice, we were also finding a remainder after dividing by 8.8.8. Each jump moves us ahead 333 dots, so jumping ahead 100100100 dots takes us ahead 3×100=3003 \times 100 = 3003×100=300 dots. Then 37×8=296,37 \times 8 = 296,37×8=296, so the remainder when we divide 300300300 by 888 is 300−296=4.300 - 296 = 4.300−296=4. Therefore, the 100 th\mathit{100^\textit{ th}}100 th segment takes us to dot 4.\mathit{4}.4.

Can you use a similar idea to answer today's challenge?

    def frog(dot_position, total_dots):
    n = 0

    while (5 * n) % total_dots != dot_position:
        n += 1

    return n

print(frog(1, 16)) # 13

    """
solution of:
https://brilliant.org/daily-problems/circle-star-1/
"""

import math

def extended_gcd(in_a, in_b):
    """returns (g, x, y) such that in_a*x + in_b*y = g = gcd(in_a, in_b)"""
    x_0, x_1, y_0, y_1 = 0, 1, 1, 0
    while in_a != 0:
        (cur_q, in_a), in_b = divmod(in_b, in_a), in_a
        y_0, y_1 = y_1, y_0-cur_q*y_1
        x_0, x_1 = x_1, x_0-cur_q*x_1
    return in_b, x_0, y_0

def get_number_of_jumps(start_pos, goal_pos, number_of_dots, jump_len):
    """
    finds the minimal number of jumps needed to reach the goal_pos from start_pos
    """
    if start_pos == goal_pos:
        jump_number = 0
    else:
        gcd_val, inv_jump_len, _ = extended_gcd(jump_len, number_of_dots)
        if gcd_val == 1:
            jump_number = ((goal_pos-start_pos)*inv_jump_len)%number_of_dots
        else:
            jump_number = math.inf
    return jump_number

FROG_DATA = {'A': 16, 'B': 17, 'C': 18, 'D': 19}
OPTIMAL_FROG = ''
MIN_VAL = math.inf
for cur_frog in FROG_DATA:
    cur_val = get_number_of_jumps(0, 1, FROG_DATA[cur_frog], 5)
    if cur_val < MIN_VAL:
        OPTIMAL_FROG = cur_frog

print(f"Answer: {OPTIMAL_FROG}")
