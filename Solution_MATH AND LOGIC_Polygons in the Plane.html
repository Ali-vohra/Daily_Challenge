Polygons in the Plane

What is the maximum number of regions that two identical regular quadrilaterals (squares) can divide a plane into?

In the image above, we see that it is possible to use two squares to divide the plane into 444 regions (including the exterior of both squares), but is that the maximum? If not, how might we increase the number of regions?

For a look at a strategy to maximize the number of regions, keep reading. Or, jump to todayâ€™s challenge for the same problem with polygons having many more sides.

We hope to establish a pattern to be able to solve the challenge problem, so letâ€™s start by taking a step back and work on dividing the plane with two triangles. In the image below, we see that we can also create 444 regions with two identical regular triangles.

It seems that if we rotate one of the triangles, we might be able to create more regions. Suppose we start with both triangles occupying the same space, one on top of the other. If we rotate one of them a bit, a vertex and some area will peek out.

By the symmetry of the regular triangle, if rotating causes a vertex and some area to poke out at one corner of the triangle, then the same will happen at all three corners. So we have one region for each corner of each triangle, as well as their shared interior and their shared exterior, for a total of 888 regions.

Why might this be the maximum number of regions? Informally, excluding the interior and exterior regions formed by the two triangles (there can only be one of each), a region must include at least one vertex of at least one triangle. So, by minimizing the number of triangle vertices in each region, we maximize the number of regions. In the arrangement above, every non-interior/exterior region includes only one vertex, so we have formed the most regions possible.

So letâ€™s go back to the squares and try the same strategy. We place one on top of the other, and rotate one so that a corner peeks out.

Again, by the symmetry to the square, one corner peeking out means all four corners peek out. We have one region for each corner of each square, as well as their shared interior and exterior, for a total of 101010 regions.

If we do the same with a pentagon, we get 121212 regions.

If you couldnâ€™t possibly draw the polygons accurately, could you follow this pattern to determine the maximum number of regions? Unless you can draw a regular polygon with 100100100 sides, thatâ€™s todayâ€™s challenge.

For a pair of identical regular polygons, you can rotate one of the polygons in such a way that all the vertices (corners) of both polygons are only ever used once in any region.

Doing this creates small triangular regions at all the vertices of both polygons. So if we have 2 polygons with nnn vertices, this would give us 2n2n2n small regions. We also have an additional interior region and an exterior region as well. As a formula, the total number of regions is equal to 2n+22n + 22n+2, where nnn is the number of sides or vertices of the regular polygon.

For a pair of regular 100-gons, the number of regions will be:

2n+2=2(100)+2=202\Large2n + 2\newline = 2(100) + 2\newline = 202\newline2n+2=2(100)+2=202
