The goal of a Calcdoku puzzle is to fill the squares with a single digit each, with the following rules:

  1. Each of the digits 111 through 333 is used exactly once in each row and column. (There is no region rule as in Sudoku.)

  2. Any marked region indicates a number and an operation (+,−,×,(+,-,\times,(+,−,×, or ÷).\div).÷). The operation applies to digits in the region, and when applied to the digits in some order, they result in the given number. In the example below, the region marked “ 7+"“\, 7+"“7+" must be filled with three digits selected from 111 through 333 that have a sum of 7.7.7. Digits can be repeated in a marked region, but not in a row or column.

def check_if_valid(grid): # Must be edited based on the question.
    if grid[0] + grid[4] + grid[8] + grid[12] + grid[5] != 11:
        return False

    if grid[1] * grid[2] * grid[6] != 9:
        return False

    if grid[3] * grid[7] * grid[11] * grid[15] * grid[14] != 48:
        return False

    if grid[9] + grid[10] + grid[13] != 10:
        return False

    return True

import itertools

def check_conditions(in_data):
    """
    Verifies, if in_data (storing only the first n-rows) satisfies
    the "arithmetic" conditions of the puzzle
    https://brilliant.org/daily-problems/calcdoku-7/
    (for which only the first n-rows are needed).
    If the input data has the size n, it assumes,
    that the result of this function for the first n-1 "rows" was True.
    """
    res = True
    if len(in_data) == 2:
        if in_data[0][1]*in_data[0][2]*in_data[1][2] != 9:
            res = False
    elif len(in_data) == 4:
        if in_data[0][0]+in_data[1][0]+in_data[2][0]+in_data[3][0]+in_data[1][1] != 11:
            res = False
        elif in_data[2][1]+in_data[2][2]+in_data[3][1] != 10:
            res = False
        elif in_data[0][3]*in_data[1][3]*in_data[2][3]*in_data[3][3]*in_data[3][2] != 48:
            res = False
    return res

def gen_solution(in_size, in_check_arithmetic_cond):
    """
    generates all Latin squares of the order in_size, which
    satisfy the additional conditions described in in_check_arithmetic_cond
    """
    def are_all_different(in_list):
        """
        checks if all of the entries in in_list are different
        """
        return len(set(in_list)) == len(in_list)
    def add_row(cur_row_data):
        if len(cur_row_data) == in_size:
            yield tuple(cur_row_data)
        else:
            for tmp_row in itertools.permutations(range(1, in_size+1)):
                tmp_row_data = cur_row_data.copy()
                tmp_row_data.append(tmp_row)
                for col_num in range(in_size):
                    cur_col = [tmp_row_data[_][col_num] for _ in range(len(tmp_row_data))]
                    if not are_all_different(cur_col):
                        break
                else:
                    if in_check_arithmetic_cond(tmp_row_data):
                        for _ in add_row(tmp_row_data):
                            yield _
    for _ in add_row([]):
        yield _

def find_reversed_rows(in_data):
    """
    finds pairs of rows, which are the same after reversing one of them
    """
    rev_rows = set()
    for (num_a, row_a) in enumerate(in_data):
        row_a_rev = tuple(reversed(row_a))
        for num_b in range(num_a+1, len(in_data)):
            row_b = in_data[num_b]
            if row_a_rev == row_b:
                rev_rows.add((num_a+1, num_b+1))
    return rev_rows


def sol_to_str(in_data):
    """
    converts a solution to a printable/readable format
    """
    raw_str = 'raw_str: {}'.format(in_data)
    sol_str = '\n'.join([' '.join([str(_) for _ in row]) for row in in_data])
    return raw_str+'\n'+sol_str+'\n\n'

ALL_REV_ROWS = set()
for cur_sol in gen_solution(4, check_conditions):
    print(sol_to_str(cur_sol))
    ALL_REV_ROWS.update(find_reversed_rows(cur_sol))

print(f"reversed rows: {ALL_REV_ROWS}")

First you need to find a place to build from. The regions will have set requirements we can use to our advantage. The 11+ and 48* regions will need a full set of 1..4 plus one extra digit. The 9* and 10+ regions can only be made using exactly 3 digits. Since they are in a corner piece we can make use of repeat digits.

Given that 9 can only be made with "331" and those 3's cannot be in the same row you now have a starting point to work with. {,3,1,} {,,3,} {,,,} {,,,_}

Given that 10 can be made from 3+3+4 or 4+4+2 we now have a chance to make another structure. The relationship of the 9 and 10 region means that the 3+3+4 option would be invalid. Leaving us with just 4+4+2.. {,3,1,} {,,3,} {,2,4,} {,4,,}

Now that we have another obvious piece that needs to be place. The second column can only accept a 1 at this point to be valid and the third column can only accept a 2 to be valid... {,3,1,} {,1,3,} {,2,4,} {,4,2,}

Since the other regions require a full column of digits, {1..4}, plus an additional digit all we need at this point is to complete the columns with the missing numbers for each row. Since the top two rows are dependent on a reflected 3,1 and 1,3 center we know that the available 4 and 2 options must also be reflected. The same principal can be applied to the last two rows as they have a reflected center with 2,4 and 4,2. {a,3,1,b} {b,1,3,a} {x,2,4,y} {y,4,2,x}

This allows us to prove that the top two rows an the bottom two rows are reflections of one another.

First, the answers: You can't reverse the order of tiles in a row twice, as that gets you the same order, and the rules don't allow that. So there can only be one or two correct answers, given the choices.

  • Any single choice
  • First and second, third and fourth
  • Second and third, first and fourth

9× jumps out as an easy piece to solve. It needs to be 3×3×1, and since the 3s must not share a row or column, the placement is certain.

10+ is next. We can't use 3 because of the 9× solution, so the only option is 4+4+2. Like the 3s before, the 4s must not be in the same row or column, so the box solves itself.

We see a pattern emerging, and tho only possible answers are "first and second" and "third and fourth."

Both of the middle columns are missing only a single tile each, so those get filled in.

That leaves the first and last column. Each of these is occupied by only a single calculation, and each must have one of each tile in it, so the math part is done.

  • The first row needs a 2 and a 4. It doesn't matter where they go.
  • The second row needs the same two tiles, but they need to be in reverse order

We can mark "The first and second" as an answer.

  • The third row needs the tiles 1 and 3, placed either way around.
  • The fourth row gets the same tiles, swapped, to fill out the columns properly.

We can mark "The third and fourth" as an answer, and be done.

There are 4 solutions to the puzzle. The first column can be any of:

  • 2, 4, 1, 3
  • 4, 2, 1, 3
  • 2, 4, 3, 1
  • 4, 2, 3, 1

The tiles in the fourth column are swapped as these are.
